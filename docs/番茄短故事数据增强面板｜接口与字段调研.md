## 5. 插件实现建议

### 5.1 请求封装

插件请求应统一封装为：

```ts
type FanqieRequestParams = {
  path: string;
  query?: Record<string, string | number | undefined>;
};
```

公共参数统一追加：

```text
aid=2503
app_name=muye_novel
```

动态安全参数处理：

- 不手动拼接 `msToken`、`a_bogus`。
- 不读取或持久化 Cookie。
- 优先在浏览器扩展环境中使用当前登录态请求官方接口。
- 如果请求被安全策略拦截，需要在真实插件环境中验证是否由页面安全 SDK 自动补齐；必要时改为 Content Script 注入页面上下文转发只读请求。

### 5.2 MVP 同步顺序

建议第一版按以下顺序拉取：

1. 调用 `/api/user/info/v2` 检测登录态。
2. 调用 `/api/author/short_article/list/v0/` 或 `/api/author/sa_stats/book_list/v0/` 获取作品列表。
3. 对每个 `book_id` 调用 `/api/author/sa_stats/single_common/v0/` 获取单篇核心数据。
4. 需要趋势时，再调用 `/api/author/sa_stats/single_by_date/v0/`。
5. 账号整体趋势用 `/api/author/sa_stats/by_date/v0/`。

### 5.3 请求频率

建议保守设置：

| 项 | 建议值 |
| --- | --- |
| 单次并发 | 2 |
| 单篇请求间隔 | 300ms - 800ms |
| 全量同步触发 | 用户手动触发优先 |
| 自动同步 | MVP 暂不默认开启 |
| 失败重试 | 网络错误最多 1 次，登录/权限错误不重试 |
