## 状态说明

| 状态 | 含义 |
| --- | --- |
| 待开始 | 尚未进入开发 |
| 进行中 | 正在开发或验证 |
| 待确认 | 需要补充接口、产品或验收信息 |
| 已完成 | 已开发并通过验收 |
| 阻塞 | 有外部条件导致无法继续 |

## 建议执行顺序

1. 先完成 M0，确认接口和字段是否足够支撑 MVP。
2. 再完成 M1，保证插件能安装、打开、通信。
3. 接着完成 M2，优先跑通「拉取数据 -> 标准化 -> 入库」。
4. 然后完成 M3，让用户能真正看到榜单、排序、筛选和导出。
5. 最后按运营价值逐步推进 M4 和 M5。

## 跟踪建议

- 每个任务开发前，将状态从「待开始」改为「进行中」。
- 遇到官方接口字段不确定时，将状态改为「待确认」。
- 每个任务完成后，必须按「验收标准」检查，再改为「已完成」。
- MVP 阶段优先处理 P0，P1 只做影响可用闭环的任务，P2 延后。
